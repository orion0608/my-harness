# /lesson-record [问题描述]

记录开发过程中有意义的问题到经验捕获清单。

## Agent 行为

1. 读取 `lesson-capture` 规则，获取「有意义」判断标准与操作流程
2. 确定待记录问题：
   - 若用户提供 `[问题描述]`：结合当前会话上下文扫描与理解，评估是否符合记录标准
   - 若未提供：回顾当前会话，识别符合标准的有意义问题
3. 向用户展示候选问题及符合的维度（上下文浪费 / 生产效率 / 代码质量 / 部署效率），用 AskQuestion 确认是否记录及待记录项
4. 用户确认后，按 `lesson-capture` 规则处理
5. 输出记录结果（编号、描述、当前次数；若触发提炼则告知后续动作）
