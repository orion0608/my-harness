## 章节要求

### 概述

- 第一句必须让读者在 10 秒内理解"这是什么项目"
- 用具体功能而非抽象描述
  - ✅ "一个支持 Markdown 预览的笔记应用"
  - ❌ "一个功能强大的笔记管理系统"

### 技术栈

- 表格列出核心技术及版本约束
- **0-1 定级后必写**：「系统级别」须为 `工具级` 或 `平台级`，与分支 `requirement.md` §1 用户确认一致
- 仅列对新人理解项目必需的技术，不超过 8 项

### 快速开始

- 从 clone 到看到运行效果，步骤不超过 5 步
- 每步必须可复制执行
- 如有环境变量，标注 `.env.example` 位置

### 项目结构

- 只列一级目录，关键目录可展开到二级
- 每个目录后加一行注释说明用途

### 文档

- 链接指向 `docs/` 下的全局文档
- 按推荐阅读顺序排列

## 更新规则

| 事件 | 更新内容 |
|------|---------|
| 0-1 系统级别首次定档 | 「技术栈」表中「系统级别」行 |
| 新增模块 | "项目结构"章节 |
| 技术栈变更 | "技术栈"表格 |
| 新增全局文档 | "文档"链接列表 |
| 范围/定位变化 | "概述"章节 |
