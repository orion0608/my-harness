# version.md 编写规范

## 创建时机

项目初始化时创建。版本号变更时更新。

## 文件位置

`version.md`（项目根目录）

## 模板

```markdown
# 版本信息

> 当前版本：**x.y.z**（YYYY-MM-DD）

## 版本历史

| 版本 | 发布日期 | 类型 | 说明 |
|------|---------|------|------|
| 1.2.0 | 2026-03-01 | Minor | 新增 xxx 功能 |
| 1.1.1 | 2026-02-15 | Patch | 修复 xxx 问题 |
| 1.1.0 | 2026-02-01 | Minor | 新增 yyy 支持 |
| 1.0.0 | 2026-01-15 | Major | 首个正式发布 |

## 版本规则

本项目遵循 [语义化版本 2.0.0](https://semver.org/lang/zh-CN/)。

| 变更类型 | 版本号变化 | 说明 |
|---------|-----------|------|
| 不兼容的 API 修改 | MAJOR（x+1.0.0） | 破坏性变更，用户需要适配 |
| 向下兼容的功能新增 | MINOR（x.y+1.0） | 新功能，老用户无影响 |
| 向下兼容的问题修正 | PATCH（x.y.z+1） | Bug 修复 |

## 当前版本详情

<!-- 当前版本的变更摘要，与 CHANGELOG 中对应版本条目一致 -->
```

## 内容要求

### 版本历史

- 按日期倒序排列，最新版本在上
- 每个版本一行，信息精简——详细变更在 `CHANGELOG.md`
- "类型"列：`Major` / `Minor` / `Patch`

### 版本规则

- 固定引用 SemVer 2.0.0
- 表格说明三种版本号变化的触发条件

### 当前版本详情

- 与 `CHANGELOG.md` 中对应版本条目内容保持一致
- 仅保留最近一个版本的详情

## 更新规则

- 版本号变更时：更新"当前版本"行、追加"版本历史"行、刷新"当前版本详情"
- 不要在此文件中写入完整的变更日志——那是 `CHANGELOG.md` 的职责
- 此文件是 `CHANGELOG.md` 的**索引**，让读者快速定位版本
