# accept-log.md 编写规范

## 创建时机

实现过程中发生以下事件时追加一条记录：
- 计划中的某项被**修改**（与原计划不同）
- 计划中的某项被**拒绝/跳过**（决定不做）
- 计划外的变更被**同意**（新增需求或调整）

文件在首次事件发生时创建。

## 文件位置

`docs/<branch-id>/accept-log.md`

## 模板

```markdown
# 变更记录

> 分支：`<branch-name>`

---

## YYYY-MM-DD HH:MM

**事件类型：** <!-- 修改 / 拒绝 / 同意 -->

**涉及内容：**
<!-- 描述被修改/拒绝/同意的具体内容 -->

**原因：**
<!-- 说明为什么做出这个决定 -->

**影响：**
<!-- 对实现计划、其他任务、交付范围的影响 -->

**关联：**
<!-- 关联的 Task 编号或 commit hash -->

---
```

## 内容要求

### 事件类型

- **修改**：原计划有调整，但方向不变
- **拒绝**：原计划中的某项决定不做
- **同意**：计划外的新增内容被接受

### 涉及内容

- 精确描述变更了什么，引用原计划中的具体内容
- 格式："原计划 [X]，变更为 [Y]"

### 原因

- 必须记录**真实的决策原因**
- 不说"根据用户要求"，说"因为[具体技术/业务原因]"

### 影响

- 说明对交付范围的影响：增加了什么、减少了什么、延迟了什么
- 如果有连带影响（A 改了导致 B 也要改），一并记录

### 关联

- 关联对应的 Task 编号和 commit hash
- 方便回溯时完整还原上下文
