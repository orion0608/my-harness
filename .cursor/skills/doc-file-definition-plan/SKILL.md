# plan.md 编写规范

## 创建时机

实现计划定稿后、开始编码前创建。

## 文件位置

`docs/<branch-id>/plan.md`

## 模板

```markdown
# 实现计划：[功能名称]

> 分支：`<branch-name>`
> 创建时间：YYYY-MM-DD HH:MM
> 关联 Plan：docs/superpowers/plans/<文件名>

## 1. 目标

<!-- 一句话描述本次实现要交付什么 -->

## 2. 任务分解

### Task 1：[任务名称]

**涉及文件：**
- 新建：`path/to/file`
- 修改：`path/to/file`

**步骤：**
- [ ] 步骤 1
- [ ] 步骤 2

### Task 2：[任务名称]

<!-- 同上 -->

## 3. 依赖关系

<!-- 任务间的依赖：Task 2 依赖 Task 1 完成 -->

## 4. 涉及文件清单

| 文件 | 操作（新建/修改/删除） | 所属任务 |
|------|----------------------|---------|
|      |                      |         |

## 5. 风险点

<!-- 实现过程中需要特别注意的技术难点 -->
```

## 内容要求

### 目标

- 一句话，可验证。读完就知道做完的标志是什么

### 任务分解

- 每个 Task 对应一个可独立验证的交付单元
- 步骤粒度控制在 2-5 分钟可完成
- 步骤必须可勾选（checkbox），用于跟踪进度
- 涉及文件必须写明完整路径

### 依赖关系

- 明确 Task 之间的先后顺序
- 标注哪些 Task 可以并行

### 涉及文件清单

- 汇总所有 Task 涉及的文件，方便全局审视影响范围
- 操作类型：新建、修改、删除

### 风险点

- 标注实现中可能遇到的坑
- 如果有应对预案，一并写明
